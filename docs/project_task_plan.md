# Embodied-AI-Robotics-Industry-Research 项目任务规划

生成日期：2026-05-26

## 1. 项目定位

本项目按名称推断，定位为“具身智能与机器人产业研究”资料库与报告工程。目标不是简单收集新闻，而是沉淀一套可持续更新、可复核、可对比的行业研究体系，用于跟踪具身智能机器人在技术路线、公司竞争、商业落地、产业链和投资机会上的变化。

当前仓库为空，因此本规划先作为项目启动任务清单，后续可根据实际研究方向、目标读者和交付形式继续细化。

## 2. 研究目标

1. 建立具身智能机器人行业的基础分类框架，区分本体厂商、基础模型/平台公司、核心零部件公司、场景集成商和终端应用方。
2. 梳理国内外代表性公司、产品、技术路线、融资情况、客户合作与真实落地状态。
3. 识别产业链关键环节，包括传感器、执行器、灵巧手、关节模组、边缘计算、数据采集、仿真训练和生产制造。
4. 跟踪商业化场景，区分概念演示、试点部署、付费订单、规模化交付和可复制经营模型。
5. 输出结构化报告和可维护数据文件，形成长期更新的研究资产。

## 3. 核心研究问题

### 3.1 技术路线

- 具身智能模型如何从大语言模型、视觉语言模型发展到视觉-语言-动作模型。
- 端到端控制、模仿学习、强化学习、仿真到现实迁移、多模态感知和机器人操作数据各自的成熟度。
- 人形机器人、轮式移动机器人、机械臂、移动操作机器人在技术难点和商业化节奏上的差异。

### 3.2 竞争格局

- 海外公司：本体公司、模型平台公司、自动化场景公司分别有哪些代表玩家。
- 国内公司：人形机器人、服务机器人、工业机器人和具身智能平台公司的竞争差异。
- 评估维度：真实订单、交付能力、场景闭环、数据飞轮、硬件成本、供应链能力、团队背景和融资能力。

### 3.3 商业落地

- 哪些场景已经出现真实部署：仓储物流、制造巡检、门店零售、楼宇服务、安防巡逻、康养陪护、家庭服务等。
- 哪些场景仍以演示和试点为主，尚未形成可规模化收入。
- 单机成本、人力替代价值、运维成本、部署周期和客户付费意愿是否支持商业闭环。

### 3.4 产业链与投资

- 上游核心部件是否存在国产替代机会。
- 机器人本体厂商的成本下降路径是否清晰。
- 平台层、数据层、仿真层和应用层分别可能形成怎样的商业模式。
- 估值、融资、订单和量产声明中哪些信息需要谨慎验证。

## 6. 数据字段建议

### 6.1 公司库字段

- 公司名称
- 国家/地区
- 类型：本体厂商、模型平台、零部件、系统集成、场景运营
- 主要产品
- 目标场景
- 技术路线
- 融资阶段
- 已披露客户
- 量产/交付状态
- 可信来源链接
- 最近更新时间
- 备注与风险

### 6.2 落地案例库字段

- 案例名称
- 公司名称
- 客户/合作方
- 应用场景
- 部署地点
- 部署规模
- 是否付费
- 是否持续运营
- 证据类型：官网、财报、客户公告、媒体、视频、公司自述
- 来源链接
- 可信度评级

### 6.3 产品库字段

- 产品名称
- 公司名称
- 机器人形态
- 高度/重量
- 自由度
- 负载能力
- 续航时间
- 移动方式
- 操作能力
- 计算平台
- 价格/成本信息
- 发布日期
- 量产状态
- 来源链接

## 8. 第一轮执行清单

1. 创建 `docs/research_scope.md`，明确项目边界和术语定义。
2. 创建 `docs/source_guidelines.md`，定义引用、来源可信度和更新时间规则。
3. 创建 `data/templates/company_template.csv`、`data/templates/product_template.csv`、`data/templates/deployment_case_template.csv`。
4. 先收集 20 家国内公司和 20 家海外公司，填入初版公司库。
5. 选取 5 个重点场景，制作初版场景成熟度矩阵。
6. 从公司库中挑选 6 家代表公司，输出第一批公司深度分析。
7. 汇总第一版 `reports/industry_map.md`，形成项目的基础行业图谱。

## 9. 质量控制规则

1. 每条关键结论必须能追溯到来源链接。
2. 公司自述、媒体报道、客户公告、财报和第三方研究报告要分开标注，不能混为同等证据。
3. 对“量产”“交付”“订单”“落地”“试点”使用统一定义，避免营销表述影响判断。
4. 所有数据文件必须包含最近更新时间。
5. 报告中要明确区分事实、判断和推测。
6. 对市场规模、出货量、融资估值等高波动信息，需要标注统计口径和发布日期。
