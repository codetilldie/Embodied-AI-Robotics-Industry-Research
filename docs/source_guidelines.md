# 来源引用与可信度规则

生成日期：2026-05-26

## 1. 来源分层

| 层级 | 来源类型 | 可信度说明 | 使用方式 |
| --- | --- | --- | --- |
| A | 公司财报、上市公告、监管文件、客户公告、政府/标准组织文件 | 可信度最高，适合支撑订单、收入、采购、政策和标准结论 | 作为关键结论主证据 |
| B | 公司官网、官方新闻稿、产品页、技术博客、论文 | 适合确认产品、技术路线、合作声明和公司自述 | 需要标注“公司自述” |
| C | 权威媒体、专业行业媒体、研究机构报告 | 适合补充行业背景、融资、市场规模和第三方评价 | 需要标注发布日期和口径 |
| D | 视频、展会演示、社交媒体、访谈 | 适合记录演示能力和市场信号 | 不单独作为商业落地证据 |
| E | 无来源转载、营销号、论坛传闻、未标注发布日期内容 | 可信度低 | 原则上不用于核心结论 |

## 2. 引用字段要求

所有数据文件至少保留以下字段：

- `source_level`：A/B/C/D/E。
- `source_type`：官网、财报、客户公告、媒体、研究报告、论文、视频、社交媒体等。
- `source_url`：可访问链接。
- `source_date`：来源发布日期；无法确认时填 `unknown`。
- `last_checked`：本项目最后核验日期。
- `confidence`：高/中/低。
- `notes`：关键判断和限制。

## 3. 事实、判断、推测的区分

报告中必须显式区分三类信息：

| 类型 | 写法要求 |
| --- | --- |
| 事实 | 可以追溯到具体来源，例如公司官网、财报、客户公告 |
| 判断 | 基于多个事实得出的分析，需要说明依据 |
| 推测 | 对未来进展或商业结果的预测，需要标注不确定性 |

示例：

- 事实：某公司在官网披露机器人进入某客户场景试点。
- 判断：该场景具备商业化潜力，因为任务高频、环境半结构化、客户已有自动化预算。
- 推测：若硬件成本下降并通过安全认证，该场景有机会进入小批量交付。

## 4. 落地状态口径

| 状态 | 定义 | 证据要求 |
| --- | --- | --- |
| 概念演示 | 视频、展会、实验室 Demo | 官方视频或公开活动即可 |
| 试点验证 | 进入真实客户或真实业务环境 | 客户、场地、任务至少明确两项 |
| 小批量交付 | 多台设备或多个任务进入客户流程 | 客户公告、财报、合同、公司公告或可信媒体 |
| 规模化商用 | 多客户、多地点、持续运营 | 财报、运营数据、客户案例或长期部署证据 |

## 5. 高风险表述处理

以下表述必须谨慎：

- “全球首个”“行业第一”：除非有标准化第三方证据，否则标注为公司自述。
- “量产”：需要区分试生产、产线下线、可售产品、批量交付和稳定供应。
- “订单”：需要区分意向订单、框架协议、采购合同、付费租赁和收入确认。
- “自主完成”：需要区分全自主、半自主、遥操作、预编程和固定场景演示。
- “已落地”：需要说明是展会、试点、客户现场、运营场景还是规模化部署。

## 6. 更新机制

1. 公司库、产品库、案例库每月至少更新一次 `last_checked`。
2. 对融资、订单、量产、客户合作等高波动信息，优先使用 12 个月内来源。
3. 如果新来源与旧来源冲突，保留两者并在 `notes` 中说明差异。
4. 删除或改写关键结论时，不直接覆盖证据；应保留原来源并补充更新说明。

## 7. 文件命名建议

- 公司数据：`data/domestic_companies.csv`、`data/overseas_companies.csv`
- 产品数据：`data/products.csv`
- 落地案例：`data/deployment_cases.csv`
- 融资事件：`data/financing_events.csv`
- 公司深度分析：`reports/company_profiles/<company_slug>.md`
- 场景报告：`reports/scenario_reports/<scenario_slug>.md`
