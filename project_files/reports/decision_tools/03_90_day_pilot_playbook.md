# 战略决策工具：90 天试点打法

生成日期：2026-05-26

## 1. 试点目标

90 天试点不是为了证明机器人“很先进”，而是为了验证一个具体任务是否能形成可复制的业务价值。

## 2. 阶段安排

| 阶段 | 时间 | 任务 |
| --- | --- | --- |
| 准备 | 第 1-2 周 | 选场景、定指标、签协议、做风险评估 |
| 部署 | 第 3-4 周 | 现场改造、设备进场、员工培训 |
| 运行 | 第 5-10 周 | 按班次运行，记录数据和异常 |
| 复盘 | 第 11-12 周 | 计算 ROI，决定停止、调整或扩大 |

## 3. 必须定义的指标

- 任务成功率。
- 日均工作小时。
- 人工接管次数。
- 故障和停机时长。
- 节省工时。
- 安全事件。
- 员工接受度。
- 单次任务成本。

## 4. 试点组织

建议成立小组：

- 业务负责人：定义任务价值。
- 现场负责人：协调流程和员工。
- IT/数字化：处理系统和网络。
- 安环负责人：风险评估。
- 采购/法务：合同和责任。
- 财务：ROI 计算。

## 5. 停止条件

试点前就要写清：

- 连续两周任务成功率低于阈值。
- 出现不可接受安全事件。
- 供应商无法提供运维响应。
- 场地改造成本超过预算。
- 员工使用率过低。

## 6. 扩大条件

- 连续 4 周达成任务成功率。
- 成本收益可计算。
- 一线员工愿意使用。
- 运维责任清楚。
- 复制到第二个点位的改造成本可控。
