# 模型与平台层：机器人智能的长期入口

生成日期：2026-05-26

## 1. 平台层的意义

如果机器人本体是身体，模型和平台就是学习、开发和部署能力。长期看，机器人行业可能形成“硬件本体 + 智能平台 + 场景应用”的分工。

## 2. 平台层能力

| 能力 | 说明 | 管理层价值 |
| --- | --- | --- |
| VLA模型 | 将视觉、语言转为动作 | 提升任务泛化 |
| 仿真训练 | 在虚拟环境训练和测试 | 降低真实试错成本 |
| 遥操作数据 | 收集人类示范 | 快速学习新任务 |
| 任务编排 | 把动作组织成流程 | 接入企业 SOP |
| 设备管理 | 多机器人调度、监控、升级 | 降低运维成本 |
| 安全策略 | 限制危险动作 | 支持上线 |

## 3. 海外标杆

- NVIDIA Isaac GR00T：偏基础设施和开发生态。
- Google DeepMind Gemini Robotics：偏具身推理和模型能力。
- Physical Intelligence：偏多机器人本体和灵巧操作。
- Covariant：偏仓储抓取和商业数据闭环。
- Intrinsic：偏工业机器人开发平台。

## 4. 中国机会

中国企业拥有丰富真实场景和硬件供应链。如果能把低成本本体、训练场、真实部署数据和行业应用结合起来，有机会形成具身智能数据平台和行业机器人应用平台。

## 5. 管理层启发

普通企业不一定要自研模型，但要重视数据归属和流程沉淀。未来供应商可能更换，但企业自己的任务数据、场景改造经验、指标体系和运维经验应留下来。
